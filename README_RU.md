## Обзор

NetRuleRouter - Windows-first менеджер сетевой политики и маршрутизации трафика.

Продукт предназначен для управления тем, как трафик распределяется между уже существующими сетевыми интерфейсами, например:
- основной интернет + VPN
- провайдер A + провайдер B
- Wi-Fi + Ethernet
- Wi-Fi + VPN

NetRuleRouter не позиционируется как:
- VPN-клиент
- инструмент анонимизации
- инструмент обхода блокировок
- proxy manager
- cloud-first сервис

## Направление продукта

Текущее зафиксированное направление продукта:
- Windows-first
- local-first
- предсказуемое поведение правил
- прозрачная диагностика
- безопасный rollback
- ежедневное управление через tray
- запуск фоновой части как службы
- сильная Free-версия с понятным расширением в сторону Pro

## Бесплатная версия
Текущая бесплатная версия строится вокруг:
- ровно 2 активных маршрутов: `primary` и `secondary`
- одной активной конфигурации
- правил по приложениям
- правил по точному FQDN
- правил по поддомену / суффиксу
- правил по отдельному IP-адресу
- локального SQLite-кэша для сопоставления FQDN/IP и explain mode
- открытых текстовых пресетов, где базовый формат - YAML
- RU и EN как базовых языков локализации
- поведения Fail-Closed при недоступности `secondary`

Рекомендуемый приоритет сопоставления правил:
1. точный FQDN
2. поддомен / суффикс
3. точный IP
4. приложение
5. маршрут по умолчанию

## Технологический вектор

Планируемое направление реализации:
- `Rust` для core-логики
- `Qt` для desktop GUI

Ожидаемое разделение ответственности:
- Rust для rule engine, системной интеграции, конфигурации, кэша, логирования и диагностики
- Qt GUI для представления, взаимодействия с пользователем, интеграции с tray и связи с Rust core

Бизнес-логика по возможности должна оставаться в Rust. Граница между Rust и Qt должна быть явной, узкой и поддерживаемой.
От продукта ожидается tray-приложение Windows для повседневного управления, а фоновая часть должна запускаться вместе с Windows как служба.
